## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **앱 이름** | 오늘 뭐 먹지? |
| **패키지명** | com.todaymenu.app |
| **플랫폼** | Android 전용 |
| **최소 SDK** | API 26 (Android 8.0) |
| **타겟 SDK** | API 35 (Android 15) |
| **언어** | Kotlin |
| **빌드 도구** | Gradle (Kotlin DSL) |
| **아키텍처** | MVVM + Clean Architecture |
| **사용 환경** | 개인 사용 (메인: Galaxy Z Flip6 + 테스트: Galaxy Note 9) |
| **AI 전략** | 하이브리드 — Gemini Nano(온디바이스) + Gemini Flash(클라우드) |

> **iOS 확장 대비 설계**: Clean Architecture의 domain/data 레이어를 순수 Kotlin으로 유지하여, 향후 Kotlin Multiplatform(KMP)으로 iOS 전환 시 UI Layer(SwiftUI)만 교체하고 비즈니스 로직은 공유할 수 있도록 설계합니다.

### 1.1 핵심 가치
- 냉장고 속 재료를 실시간으로 파악
- 영수증 촬영 한 장으로 재료 자동 등록 (수동 입력도 간편하게)
- AI가 보유 재료 기반으로 유형별 메뉴 추천 + 유튜브/블로그 레시피 연결
- 유통기한 미입력 시 AI가 자동 추정, 임박 재료 알림 → 메뉴 추천 연계
- 주간/월간 식단 자동 생성
- **주부가 한 손으로도 쉽게 사용하는 직관적 한글 UI**
- 음성으로 재료를 추가하는 핸즈프리 입력 지원

### 2.2 AI / ML (무료, 하이브리드 구조)
| 기술 | 용도 | 비용 | 동작 위치 |
|------|------|------|----------|
| **Google ML Kit — Text Recognition v2** | 영수증 OCR | 무료 | 온디바이스 |
| **Google ML Kit — GenAI Prompt API** | 유통기한 추정, 영수증/음성 파싱 | 무료 | 온디바이스 (Gemini Nano) |
| **Google Gemini API (gemini-2.0-flash)** | 메뉴 추천, 식단 생성, 알림 메뉴 추천 | 무료 (일 1,500회) | 클라우드 |
| **Android SpeechRecognizer** | 음성 → 텍스트 변환 | 무료 | 온디바이스 |

> **하이브리드 AI 전략**: 단순 작업(유통기한 추정, OCR/음성 파싱)은 Gemini Nano로 온디바이스 처리하여 API 키 없이 동작합니다. 복잡 작업(메뉴 추천 3개+레시피, 주간 식단 생성, 백그라운드 알림)은 Gemini Flash 클라우드 API를 사용합니다.
> 
> **기기 요구사항**: Gemini Nano 온디바이스 기능은 Galaxy Z Flip6/Fold6 이상, Galaxy S24 이상에서 지원됩니다. 지원되지 않는 기기(Galaxy Note 9 등)에서는 모든 AI 기능이 클라우드 API로 자동 폴백됩니다.
> 
> **테스트 기기 역할**: Galaxy Note 9(Nano 미지원)은 Flash 폴백 경로를 검증하는 데 활용합니다. 메인(Z Flip6)에서는 Nano 온디바이스 경로를, 테스트(Note 9)에서는 Flash 클라우드 경로를 각각 검증하여 양쪽 모두 정상 동작을 확인합니다.

## 4. 데이터베이스 설계 (Room)

### 4.1 ingredients 테이블
```kotlin
@Entity(tableName = "ingredients")
data class IngredientEntity(
    @PrimaryKey(autoGenerate = true)
    val id: Long = 0,
    val name: String,                        // 재료명 (예: "삼겹살")
    val category: String,                    // 카테고리 (육류/채소/해산물/유제품/양념/과일/곡류·면/냉동식품/음료/기타)
    val quantity: Double = 1.0,               // 수량 (숫자, 예: 300, 2, 1.5)
    val unit: String = "개",                   // 단위 (예: "g", "ml", "개", "팩", "단", "모")
    val purchaseDate: Long,                  // 구매일 (epoch millis)
    val expiryDate: Long? = null,            // 유통기한 (선택 입력, epoch millis)
    val isExpiryEstimated: Boolean = false,  // true = AI가 추정한 유통기한 (사용자 미입력)
    val storageType: String = "fridge",      // 보관 위치: fridge / freezer / room
    val memo: String? = null,                // 메모
    val isConsumed: Boolean = false,         // 소진 여부
    val createdAt: Long = System.currentTimeMillis(),
    val updatedAt: Long = System.currentTimeMillis()
)
```

#### 화면 1: 🏠 홈 (HomeScreen)

**역할**: 앱의 대시보드, 핵심 정보를 한눈에

**레이아웃 구성**:
```
┌─────────────────────────────┐
│  오늘 뭐 먹지? 🍳     [⚙️]  │  ← 앱바
├─────────────────────────────┤
│                             │
│  🔔 알림 카드               │  ← 유통기한 임박 알림 카드
│  "구입한지 7일 지난 재료가   │     (클릭 시 해당 재료 기반
│   3개 있어요!               │      메뉴 추천으로 이동)
│   부대찌개 어떠세요?"       │
│                             │
├─────────────────────────────┤
│  📊 냉장고 현황              │
│  ←[육류][채소][해물][유제품] │  ← 카테고리별 재료 수 (10종)
│    [양념][과일][곡류][냉동]→ │     가로 스크롤 LazyRow
│    [음료][기타]              │     원형 아이콘 + 숫자
│  │ 3  │ 7  │ 2  │ 1  │...  │
├─────────────────────────────┤
│  🍽️ 오늘의 추천 메뉴        │  ← AI 추천 메뉴 1~2개
│  ┌─────────────────────┐   │     카드 형태, 탭하면
│  │ 🥘 김치찌개          │   │     레시피 상세 표시
│  │ 보유재료 4/5 사용가능 │   │
│  └─────────────────────┘   │
│                             │
│         [📷 스캔하기]        │  ← FAB 버튼 (영수증 스캔)
└─────────────────────────────┘
```

**기능 동작**:
- 앱 실행 시 자동으로 유통기한 체크 → 알림 카드 생성
- 알림 카드 클릭 → RecommendScreen으로 이동 (임박 재료 자동 선택)
- 냉장고 현황 카테고리 클릭 → FridgeScreen 해당 카테고리로 이동
- 오늘의 추천은 앱 실행 시 Gemini API 1회 호출로 생성
- FAB 클릭 → ScanScreen 이동

### 6.2 AI 유통기한 자동 추정 (EstimateExpiryUseCase)

```kotlin
/**
 * 유통기한 미입력 재료에 대해 AI가 자동으로 추정
 * 
 * 호출 시점:
 * 1. 재료 등록 시 유통기한을 입력하지 않은 경우 → 즉시 추정
 * 2. 영수증 스캔 후 일괄 등록 시 → 유통기한 없는 항목 일괄 추정
 * 
 * 동작:
 * - Gemini API에 재료명 + 보관방식을 전달
 * - 추정 결과를 expiryDate에 저장하고 isExpiryEstimated = true 설정
 * - UI에서 AI 추정값은 "~D-25 🤖" 형태로 구분 표시
 * - 사용자가 편집 아이콘(✏️)으로 직접 수정하면 isExpiryEstimated = false
 * 
 * 속도 향상 전략 (Section 9.5):
 * - 로컬 사전(expiry_defaults 테이블, 100종+)에서 먼저 조회 → 0ms 즉시 응답
 * - 사전에 없는 재료만 Gemini API 호출 (전체의 5% 미만)
 * - API로 추정한 결과도 expiry_defaults에 source="api"로 영구 저장
 *   → 같은 재료를 두 번 다시 질문하지 않음 (자기학습형 캐시)
 */

class EstimateExpiryUseCase @Inject constructor(
    private val geminiService: GeminiApiService,
    private val ingredientRepository: IngredientRepository,
    private val expiryDefaultDao: ExpiryDefaultDao  // Room 기반 로컬 사전
) {
    suspend fun estimateExpiry(ingredients: List<Ingredient>): List<Ingredient> {
        val needsEstimation = ingredients.filter { it.expiryDate == null }
        
        // Step 1: Room expiry_defaults 테이블에서 조회 (100종+ 로컬 사전)
        val localResults = mutableListOf<Ingredient>()
        val needsApi = mutableListOf<Ingredient>()
        
        for (item in needsEstimation) {
            val default = expiryDefaultDao.find(item.name, item.storageType)
            if (default != null) {
                // 로컬 사전 히트 → 즉시 적용, API 호출 없음
                localResults.add(item.copy(
                    expiryDate = item.purchaseDate + (default.estimatedDays * 86400000L),
                    isExpiryEstimated = true
                ))
            } else {
                needsApi.add(item)
            }
        }
        
        // Step 2: 로컬 사전에 없는 항목만 Gemini API 배치 호출 (대부분 0건)
        if (needsApi.isNotEmpty()) {
            val prompt = buildExpiryEstimationPrompt(needsApi)
            val response = geminiService.generateContent(prompt)
            val apiResults = parseGeminiResponse(response)
            
            // Step 3: API 결과를 로컬 사전에 영구 저장 (자기학습형 캐시)
            apiResults.forEach { result ->
                expiryDefaultDao.insert(ExpiryDefaultEntity(
                    ingredientName = result.name,
                    storageType = result.storageType,
                    estimatedDays = result.estimatedDays,
                    source = "api"  // Gemini에서 학습한 데이터 표시
                ))
            }
        }
        
        return localResults + apiResults
    }
}
```

### 6.6-1 Gemini 응답 JSON 파싱 방어 (GeminiResponseParser)

```kotlin
/**
 * Gemini API(Flash/Nano 모두)의 응답을 안전하게 JSON으로 파싱
 * 
 * [문제]
 * Gemini는 JSON을 요청해도 다음과 같이 반환하는 경우가 빈번함:
 * - 마크다운 코드블록으로 감싸기: ```json\n{...}\n```
 * - 앞뒤에 설명 텍스트 추가: "네, 다음은 결과입니다.\n{...}"
 * - 불완전한 JSON (중괄호 미닫힘, 후행 쉼표 등)
 * - 빈 응답 또는 safety filter 차단
 * 
 * [해결 전략]
 * 1. 전처리: 코드블록/설명 텍스트 제거 → 순수 JSON 추출
 * 2. 파싱 시도: Kotlin Serialization으로 파싱
 * 3. 실패 시 재시도: 최대 2회, "순수 JSON만 응답하세요" 강조 프롬프트로 재호출
 * 4. 최종 실패 시: 사용자에게 "다시 시도해 주세요" 안내
 */

object GeminiResponseParser {

    /**
     * Step 1: 응답 텍스트에서 순수 JSON만 추출
     */
    fun extractJson(rawResponse: String): String {
        var cleaned = rawResponse.trim()
        
        // 마크다운 코드블록 제거: ```json ... ``` 또는 ``` ... ```
        val codeBlockRegex = Regex("""```(?:json)?\s*\n?([\s\S]*?)\n?\s*```""")
        val codeBlockMatch = codeBlockRegex.find(cleaned)
        if (codeBlockMatch != null) {
            cleaned = codeBlockMatch.groupValues[1].trim()
        }
        
        // 앞뒤 비-JSON 텍스트 제거: 첫 번째 { 또는 [ 부터 마지막 } 또는 ] 까지
        val jsonStartIndex = cleaned.indexOfFirst { it == '{' || it == '[' }
        val jsonEndIndex = cleaned.indexOfLast { it == '}' || it == ']' }
        if (jsonStartIndex >= 0 && jsonEndIndex > jsonStartIndex) {
            cleaned = cleaned.substring(jsonStartIndex, jsonEndIndex + 1)
        }
        
        // 후행 쉼표 제거 (,} → } / ,] → ])
        cleaned = cleaned.replace(Regex(",\\s*([}\\]])"), "$1")
        
        return cleaned
    }

    /**
     * Step 2: 안전한 파싱 + Step 3: 재시도 로직
     */
    suspend inline fun <reified T> parseWithRetry(
        rawResponse: String,
        retryAction: suspend () -> String,  // 재시도 시 API 재호출 람다
        maxRetries: Int = 2
    ): Result<T> {
        // 1차 시도: 전처리 후 파싱
        val cleaned = extractJson(rawResponse)
        try {
            val result = Json {
                ignoreUnknownKeys = true    // 예상 외 필드 무시
                isLenient = true            // 따옴표 누락 등 관용 처리
                coerceInputValues = true    // null → 기본값 변환
            }.decodeFromString<T>(cleaned)
            return Result.success(result)
        } catch (e: Exception) {
            // 1차 파싱 실패 → 재시도
        }
        
        // 2차~3차 시도: "순수 JSON만" 강조하여 재호출
        repeat(maxRetries) { attempt ->
            try {
                val retryResponse = retryAction()
                val retryCleaned = extractJson(retryResponse)
                val result = Json {
                    ignoreUnknownKeys = true
                    isLenient = true
                    coerceInputValues = true
                }.decodeFromString<T>(retryCleaned)
                return Result.success(result)
            } catch (e: Exception) {
                if (attempt == maxRetries - 1) {
                    return Result.failure(e)  // 최종 실패
                }
            }
        }
        return Result.failure(Exception("JSON 파싱 최종 실패"))
    }
}

// 사용 예시 (UseCase에서)
val result = GeminiResponseParser.parseWithRetry<MenuRecommendationResponse>(
    rawResponse = apiResponse.text,
    retryAction = {
        // 재시도 시 프롬프트에 강조 문구 추가

    }
)
```

### 6.7 "요리 완료" 재료 자동 차감

```kotlin
/**
 * 사용자가 레시피 상세에서 [요리 완료] 버튼을 누르면:
 * 1. 사용 재료별로 "전부 사용 / 일부 사용" 선택 다이얼로그 표시
 * 2. 전부 사용 → isConsumed = true (소진 처리)
 * 3. 일부 사용 → quantity 차감 (예: 300g 중 200g 사용 → 100g 잔여)
 * 4. 차감 후 quantity ≤ 0 이면 자동 소진 처리
 * 5. menu_history에 요리 기록 저장
 * 
 * UI: 각 재료 옆에 [전부] [일부] 버튼
 *     [일부] 선택 시 → 사용량 입력 필드 표시 (숫자 키패드)
 *     기본값: 레시피에 명시된 사용량 (AI가 제공)
 */
suspend fun completeCooked(menu: MenuRecommendation, usageMap: Map<Long, Double>) {
    usageMap.forEach { (ingredientId, usedAmount) ->
        val ingredient = ingredientRepository.getById(ingredientId) ?: return@forEach
        val remaining = ingredient.quantity - usedAmount
        if (remaining <= 0) {
            // 전부 소진
            ingredientRepository.update(ingredient.copy(isConsumed = true, quantity = 0.0))
        } else {
            // 부분 차감
            ingredientRepository.update(ingredient.copy(quantity = remaining))
        }
    }
    menuHistoryRepository.save(menu.toHistoryEntity())
}

### 9.2 Gemini Nano 온디바이스 연동 (ML Kit Prompt API)

```kotlin
/**
 * Galaxy Z Flip6에 탑재된 Gemini Nano를 ML Kit Prompt API로 호출
 * 
 * 특징:
 * - API 키 불필요, 완전 무료, 오프라인 동작
 * - 데이터가 기기 밖으로 나가지 않음 (프라이버시)
 * - 포그라운드에서만 동작 (WorkManager 백그라운드 불가)
 * - Alpha 단계이므로 복잡한 JSON 생성보다 단순 작업에 적합
 * 
 * 적용 기능:
 * 1. 유통기한 추정 — 로컬 사전(100종) 미스 시 Nano로 추정
 * 2. 영수증 OCR 텍스트 → 재료 목록 파싱
 * 3. 음성 인식 텍스트 → 재료+수량 파싱
 * 
 * 폴백: Nano 미지원 기기(테스트 기기 등)에서는 Flash API로 자동 전환
 */

// AI 엔진 라우터 — Nano 가능 여부에 따라 자동 분기
class AiEngineRouter @Inject constructor(
    private val nanoService: GeminiNanoService,
    private val flashService: GeminiFlashService
) {
    suspend fun parseIngredients(text: String): Result<List<ScannedIngredient>> {
        return if (nanoService.isAvailable()) {
            nanoService.parseIngredients(text)  // 온디바이스
        } else {
            flashService.parseIngredients(text)  // 클라우드 폴백
        }
    }
    
    suspend fun estimateExpiry(name: String, storage: String): Result<Int> {
        return if (nanoService.isAvailable()) {
            nanoService.estimateExpiry(name, storage)
        } else {
            flashService.estimateExpiry(name, storage)
        }
    }
}

// Gemini Nano 서비스 (ML Kit Prompt API)
class GeminiNanoService @Inject constructor(
    @ApplicationContext private val context: Context
) {
    private val generativeModel = GenerativeModel.newBuilder()
        .setPromptType(PromptType.UNSPECIFIED)
        .build()
    
    suspend fun isAvailable(): Boolean {
        return generativeModel.checkStatus() == FeatureStatus.AVAILABLE
    }
    
    suspend fun parseIngredients(ocrText: String): Result<List<ScannedIngredient>> {
        val prompt = """
            <INSTRUCTIONS>
            한국 마트 영수증 텍스트에서 식재료만 추출하세요.
            봉투, 할인, 합계 등은 제외하세요.
            JSON 배열로만 응답하세요. 다른 텍스트는 포함하지 마세요.
            형식: [{"name":"재료명","quantity":수량,"unit":"단위"}]
            </INSTRUCTIONS>
            <INPUT>$ocrText</INPUT>
        """.trimIndent()
        val response = generativeModel.generateContent(prompt)
        return parseJsonResponse(response.text)
    }
}
```

### 9.3 Gemini Flash 클라우드 API 키 관리 (local.properties)

/**
 * 개인 사용 앱이므로 API 키를 local.properties에 저장하고
 * BuildConfig를 통해 빌드 시 자동 주입합니다.
 * 
 * 장점:
 * - 앱 내 API 키 입력 UI 완전 제거 (온보딩 허들 0)

 * 1. Google AI Studio(https://aistudio.google.com/)에서 Gemini API 키 발급
 * 2. Pexels(https://www.pexels.com/api/)에서 이미지 API 키 발급 (무료 회원가입)

 *    PEXELS_API_KEY=your_pexels_api_key_here
 * 4. 빌드 시 자동으로 BuildConfig.GEMINI_API_KEY, BuildConfig.PEXELS_API_KEY로 접근
 */

// app/build.gradle.kts
android {
    defaultConfig {
        // local.properties에서 API 키 읽어 BuildConfig에 주입
        val properties = java.util.Properties()
        val localPropertiesFile = rootProject.file("local.properties")
        if (localPropertiesFile.exists()) {
            properties.load(localPropertiesFile.inputStream())
        }
        buildConfigField(
            "String",
            "GEMINI_API_KEY",
            "\"${properties.getProperty("GEMINI_API_KEY", "")}\""
        )
    }
    buildFeatures {
        buildConfig = true
    }
}

// 사용 시
val apiKey = BuildConfig.GEMINI_API_KEY

### 13.3 에러 대응 가이드
- Gemini Flash API 호출 실패 시 → "인터넷 연결을 확인해 주세요" + 재시도 버튼 + 캐시 결과 표시
- Gemini Nano 미지원 기기 → 해당 기능을 Flash API로 자동 폴백 (사용자에게 안내 불필요)
- Gemini Nano BUSY 에러 (과다 호출) → 잠시 대기 후 재시도 (exponential backoff)
- OCR 인식 실패 시 → "인식이 어려워요. 직접 입력해 주세요" + 수동 입력 전환
- 음성 인식 실패 시 → "잘 알아듣지 못했어요. 다시 말씀해 주세요" + [다시 시도] / [직접 입력] 버튼
- 음성 인식 권한 거부 시 → "마이크 사용 권한이 필요해요" + 설정 이동 안내
- 네트워크 없음 → Nano 기반 기능(재료 관리, 유통기한 추정, 스캔/음성 파싱)은 정상 동작. 메뉴 추천/식단 생성만 비활성화 + 캐시 결과 표시
